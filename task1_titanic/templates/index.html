📖 About This Project

This project was built as part of a Machine Learning internship at CodSoft. The goal was to predict whether a passenger aboard the RMS Titanic would survive the tragedy of April 1912 — one of the most iconic binary classification problems in data science.

Project Pipeline

📥 Load Dataset 🔍 EDA & Visualisation 🧹 Data Cleaning ⚙️ Feature Engineering 🤖 Model Training 📊 Evaluation 💾 Export & Deploy

We started by exploring the raw Titanic dataset — visualising survival distributions across gender, passenger class, age, and embarkation port. Columns like PassengerId, Name, Ticket and Cabin were dropped as they carried no predictive signal. A new feature, Family Size, was engineered from SibSp + Parch + 1 to better capture group survival dynamics.

Challenges We Solved

  • Missing Age values — imputed with the column mean to preserve all rows rather than discarding them.
  • Missing Embarked values — the two affected rows were dropped as they were too few to impute reliably.
  • Categorical encoding — Sex (male→0, female→1) and Embarked (C→0, Q→1, S→2) were label-encoded so the model could process them numerically.
  • Feature redundancy — SibSp and Parch were combined into a single Family Size feature and the originals removed, reducing noise.
  • Model selection — Logistic Regression, Decision Tree, Random Forest, KNN, and Gradient Boosting were all tested. Gradient Boosting delivered the highest accuracy and was chosen for deployment.

The trained sklearn pipeline was serialised with pickle and served through a Flask REST API, making live predictions accessible via this interactive UI.
